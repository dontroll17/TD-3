# TD-3 Glitch Token Generator

**Превращает аномальные токены больших языковых моделей в музыкальный глитч-поток для синтезатора Roland (ДА-ДА) TD-3.**

## 🎯 Описание проекта

Этот скрипт реализует уникальный подход к музыкальной генерации: он преобразует глитч-токены (аномальные данные) из больших языковых моделей в реальное звучание через MIDI-протокол, управляя синтезатором Roland TD-3.

Основная идея — использовать математическую структуру **19-ТЭТ** (19-ступенчатая равномерная темперация) для точного маппинга числовых значений токенов в музыкальные параметры: высоту тона, динамику, слайд и акценты.

---

## 🎛️ Возможности

- 🔄 **Конвертация токенов в музыку**: Любой числовой ID токена превращается в MIDI-ноту
- 🎨 **19-ТЭТ темперация**: Точная настройка высоты тона с шагом ~63.16 центов
- 🎯 **Умная динамика**: Автоматическое определение акцентов на основе свойств токена
- 🎵 **Скольжение (Slide)**: Автоматическое включение скольжения при чередовании четных/нечетных токенов
- 🧮 **Pitch Bend компенсация**: Точный сдвиг высоты тона для достижения 19-ТЭТ интервалов
- 🎮 **Настройка BPM**: Гибкая регулировка скорости генерации паттерна

## 🚀 Быстрый старт

### 1. Подключение TD-3 к компьютеру

1. Подключите Roland TD-3 к компьютеру через USB-MIDI кабель
2. Установите драйверы Roland (если требуется)
3. Проверьте доступные MIDI-порты:

```python
import mido
print(mido.get_output_names())
```

### 2. Настройка скрипта

Отредактируйте файл `Input.py`:

```python
# Укажите точное имя вашего MIDI-порта TD-3
MIDI_PORT_NAME = 'Ваш_MIDI_Порт_TD-3'  # Замените на имя из шага 1

# Настройте базовую ноту (оптимально 36 = До малой октавы)
BASE_NOTE = 36

# Настройте BPM (по умолчанию 135)
# Для плотного индастриала попробуйте 140-160
# Для более расслабленного звучания попробуйте 100-120
```

### 3. Запуск генератора

```bash
python Input.py
```

Скрипт начнет бесконечную генерацию паттерна на основе массива `GLITCH_TOKENS`.

Для остановки нажмите `Ctrl+C`.

## 🎹 Принцип работы

### 1. Математика 19-ТЭТ

```
1 шаг 19-ТЭТ = 1200 центов / 19 ≈ 63.158 центов
```

Скрипт вычисляет:
- Ближайшую стандартную 12-ТЭТ ноту
- Остаток в центах, который компенсируется Pitch Bend

### 2. Карта токенов в музыку

| Параметр | Алгоритм | Результат |
|----------|----------|-----------|
| **Высота тона** | `token % 38` (2 октавы в 19-ТЭТ) | Нота + Pitch Bend |
| **Динамика (Accent)** | `token % 3 == 0` или `token > 50000` | Velocity 127 (Accent) или 60-85 |
| **Скольжение** | `next_token % 2 == 0` | Slide включен или выключен |

### 3. MIDI-команды

- `pitchwheel` — установка Pitch Bend для 19-ТЭТ точности
- `note_on` — запуск ноты с определенной динамикой
- `note_off` — завершение ноты (для staccato)
- `control_change` (CC#123) — сброс всех нот при остановке

## 🛠️ Настройка

### Настройка базовой ноты

```python
BASE_NOTE = 36  # До малой октавы (C2)
```

| Значение | Нота | Октава |
|----------|------|--------|
| 24 | До (C) | 0 |
| 36 | До (C) | 1 (малая) |
| 48 | До (C) | 2 (большая) |
| 60 | До (C) | 3 (малая октава) |

### Настройка диапазона Pitch Bend

```python
PITCH_BEND_RANGE_CENTS = 200.0  # ±2 полутона (стандарт TD-3)
```

Если ваш синтезатор настроен на другой диапазон (например, ±12 полутонов), измените это значение.

### Изменение массива токенов

Замените массив `GLITCH_TOKENS` на свои данные из LLM:

```python
GLITCH_TOKENS = [
    9345, 23049, 1204, 88342, 4032,  # ... ваши токены
]
```

## 🎨 Примеры использования

### 1. Прямое использование GLITCH_TOKENS

Скрипт поставляется с предустановленным массивом аномальных токенов:

```python
GLITCH_TOKENS = [
    9345, 23049, 1204, 88342, 4032, 99231, 1024, 456, 88342, 
    23049, 99231, 7763, 12, 4096, 65536, 100234, 432, 889
]
```

### 2. Интеграция с LLM

Создайте скрипт для получения токенов из вашей LLM:

```python
from transformers import AutoTokenizer
import Input

# Загрузка модели
tokenizer = AutoTokenizer.from_pretrained("meta-llama/Llama-2-7b")
model_input = "Ваш промпт здесь"
tokens = tokenizer.encode(model_input)

# Запуск с полученными токенами
Input.GLITCH_TOKENS = tokens
```

### 3. Генерация случайных токенов

```python
import random
Input.GLITCH_TOKENS = [random.randint(0, 100000) for _ in range(50)]
```

## 🐛 Устранение неполадок

### Ошибка: "Не удалось найти MIDI-порт"

**Решение**: Проверьте точное имя порта:

```python
import mido
print(mido.get_output_names())
```

Убедитесь, что `MIDI_PORT_NAME` в точности совпадает (с учетом регистра и кириллицы).

### TD-3 не реагирует на звук

**Проверьте**:
1. ✅ TD-3 включен и подключен к компьютеру
2. ✅ Выбран правильный MIDI-порт
3. ✅ Громкость TD-3 не выключена
4. ✅ На TD-3 выбрана鼓音色 (например, Kick, Snare)

### Нет Pitch Bend эффекта

**Проверьте настройки TD-3**:
- Убедитесь, чтоPitch Bend Range настроен на ±2 полутона
- Убедитесь, чтоPitch Bend sensitivity не выключен
